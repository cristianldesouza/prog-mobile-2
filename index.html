Progressive Web Application (PWA)

05 Mar 2020

PWA. Repensando seu produto e recuperando os valores da web.

O PWA nada mais é do que uma página web, é desenvolvida usando tecnologias web como HTML, CSS e JavaScript. Isso simplifica o desenvolvimento de aplicativos (não necessariamente mobile), ou seja, é um website desenvolvido para parecer um aplicativo instalado no aparelho.

Os PWA’s utilizam service workers, que são scripts desenvolvidos em JavaScript que controlam o armazenamento de cache, isso permite um desempenho mais alto, pois permite que grande parte dos dados persistam no dispositivo. Esses service workers em alguns casos permitem até que o web app funcione offline.

Entre os service workers disponíveis, temos:

  • Notificações;
  • Ícones de acesso;
  • Processos rodando em background;
  • Modo offline;
  • Acesso a câmera;
  • Galeria de fotos;
  • Microfone;
  • Geolocalização;
  • Contatos.

Vantagens do PWA:

- Benefícios para as empresas e desenvolvedores: Como os PWA’s só dependem de CSS3 e JavaScript basicamente, gerar uma nova atualização para todas as plataformas é muito mais simples;
- Reutilização do mesmo código para plataformas diferentes;
- Independência das App Stores pode ser uma vantagem em um certo sentido, pois muitos usuários acabam tendo certa resistência para instalar apps.

Com o lançamento do Chrome 70 os PWA’s conseguem rodar no windows como aplicativos comuns, podendo acessar notificações, a cortana e as live tiles (blocos de notificações dinamicas do windows 10). Para macOS e linux, no Chrome, o suporte deverá chegar ao mercado a partir da versão 72.

Grandes empresas como Microsoft e Google são totalmente pró PWA, por aumentarem significamente a quantidade de App’s disponíveis.

Já a Apple ainda resiste a essa tecnologia, o safari não permite notificações nem funcionamento offline, entre outros services workers.

PaperPlanesWorld

09 Mar 2020

Use seu celular para lançar e capturar aviões de papel pelo mundo todo!

PaperPlanes.world é um website desenvolvido pela empresa californiana Active Theory. Foi desenvolvido para mostrar o poder do PWA, rodando como um aplicativo nativo nos navegadores dos celulares.

Quando o app é aberto em um computador, temos apenas umas página, que diz para aproveitar a experiência no smartphone.

No smpartphone é onde a mágica acontece, o usuário pode criar aviões de papel, carimbar o papel com sua localização e jogá-lo ao “vento” onde outras pessoas ao redor do mundo podem capturá-lo e carimbar com a sua localização também.

Mas, como isso é possível?

Por mais complexo que pareça esta façanha, no projeto utiliza basicamente JavaScript e WebGL.

WebGL é uma API que oferece suporte para renderização gráfica a partir do HTML 5.

Para entrada e saida de dados, foi utilizada uma rede de serviçoçs do Google Cloud Platform. Foi utilizado geocodigicação no cabeçalho das requisições para obter o carimbo com a localização do usuário.

Os dispositivos são conectados através do Socket.IO, afunilando todas as mensagens para o WebGL renderizar os aviões.

Curiosidades

Este projeto foi divulgado no Google I/O 2016 em 18 de maio de 2016.

No dia do evento, haviam mais de 7000 pessoas criando aviões e "arremessando-os" ao vento, um enorme telão no centro do palco era atualizado com os aviões arremessados em tempo real.

Mais de 40 mil aviões foram jogados em meia hora de apresentação, hoje, em 2020, temos mais de 5 milhões de aviões no no app.

Este projeto abriu as portas para os PWA, mostrando sua força, utiliza vibracal, sensores de localização e geolozaliação dos aparelhos.
